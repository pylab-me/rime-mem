# rime-mem

让 Rime 拥有更聪明的“记忆”与联想能力。

> 项目 setup 的动机，之前见到有比较多的讨论指出，想要「[上下文调频功能](https://github.com/rime/librime/issues/897)」。
> 
> 嗯，我做了一个 MVP 的版本，应该是比 MVP 更加走前一步，我已经在本地使用了一段时间，确保 `rime-mem` 不是一个玩具。

## 它能做什么

在日常输入中，我们经常会遇到这些场景：

- 刚输入过的词，希望接下来能优先出现
- 正在写一段内容，输入法能根据上下文给出更合理的候选
- 用久了，输入法能“记住”自己的用词习惯

`rime-mem` 就是为了让 Rime 在这些场景下表现更好而设计的。

它不是一个独立输入法，而是为 Rime 增加的一层“记忆增强”能力。

## 主要功能

**记忆输入习惯**  
不只是查静态词库，而是会根据你的历史输入，让常用词更容易被选到。

**理解上下文**  
在连续输入时，能结合前面打过的内容，给出更符合当前语境的候选词。

**长期使用越用越顺**  
使用时间越长，对个人用词习惯的“记忆”就越准确。

**保持流畅体验**  
所有这些增强能力，都在后台运行，不影响输入本身的流畅度。

## 适合谁用

- 觉得 Rime 默认的候选不够“懂你”
- 希望输入法能记住自己的用词习惯
- 在连续输入场景（如写作、聊天）希望有更顺畅的体验
- 愿意尝试让输入法变得更聪明一点

## 简单说明

这个项目把“记忆”相关的能力做成了一个独立模块，不干扰 Rime 本身的稳定和灵活。你只需要正常使用输入法，就能逐渐感受到候选词变得更贴合自己的习惯。
